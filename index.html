Answers to 3 questions.

What is git workflow?

Git workflow is a module that enables countless members to work on one project.

Why do we need git workflow?

Git workflow is letting all team members to easily work on single project.

How does git workflow work?

The team leader creates a project and invites his team members so they can work on it, but they only can change something in their respective branches and in the end only team leader can decide if changes will be added to the main branch.

What is git rebase?

Git rebase is an option that will "merge" the whole base of branch into another by rewriting the main where the branch was created.

What is the difference between Rebase and Merge?

The difference is that Merge is safer option compared to rebase due to it's "merging" only the results of 2 branches into one while Rebase is "merging" the whole branch into main and removing that branch.

What is a git fetch?

Git fetch is an command that downloads contents from repository.

What is a difference between Fetch and Pull?

Fetch will only download the contents from repository while Pull will not only download but will also change everything in your files if there is some changes to download.
